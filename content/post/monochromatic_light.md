## 为什么只看 RGB 值不够

显示屏输出的每一个颜色，本质上都对应一条光谱曲线。相同的 sRGB 数值在不同背光、发光材料和滤光结构下，可能给出不同的窄峰宽度、肩峰和角度色偏。人眼最后看到的颜色是 SPD 与标准观察者函数共同作用的结果。

> 工程上最容易踩坑的地方是：截图里的颜色值很稳定，但实测白点和视角色偏并不稳定。

把光谱转换到 CIE XYZ，通常写作：

$$
X = k\int P(\lambda)\overline{x}(\lambda)d\lambda
$$

$$
Y = k\int P(\lambda)\overline{y}(\lambda)d\lambda
$$

$$
Z = k\int P(\lambda)\overline{z}(\lambda)d\lambda
$$

## 测量链路

一套比较朴素的实验链路如下。它不追求一步到位，但每个环节都可以检查，适合写入自动化测试脚本。

1. 显示固定灰阶或白场，关闭动态对比度、环境光补偿和自动亮度。
2. 光谱仪暗电流校正后，对同一中心点重复采样至少三次。
3. 用标准观察者函数积分得到 XYZ，再计算 $x,y,u',v'$。
4. 对比目标白点，例如 D65，记录 $\Delta u'v'$ 或 JNCD。

```python
import numpy as np

def xyz_from_spd(wavelength, power, cmf):
    step = np.gradient(wavelength)
    xyz = np.sum(power[:, None] * cmf * step[:, None], axis=0)
    return xyz / xyz[1]

def xy_from_xyz(xyz):
    total = np.sum(xyz)
    return xyz[0] / total, xyz[1] / total
```

## 白点偏差怎么写进报告

白点偏差推荐同时给出数值和解释。比如目标为 D65，报告里写出 $x,y$，再补充 $\Delta u'v'$ 或 JNCD，比只给“偏蓝”“偏绿”更可复现。

| 项目 | 目标 | 实测 | 备注 |
| --- | ---: | ---: | --- |
| 亮度 | 500 cd/m² | 493 cd/m² | 中心点 |
| 白点 x | 0.3127 | 0.3098 | 轻微偏蓝 |
| 白点 y | 0.3290 | 0.3234 | 需复测角度项 |

## 几个常见坑

如果只做一次中心点测量，很容易把局部现象当成屏幕整体特性。更稳妥的做法是把测量点、驱动状态、曝光时间、积分时间、环境温度和仪器校准日期都写下来。

- 低灰阶测量时，自动曝光可能改变噪声结构。
- 窄峰光源对光谱仪带宽和杂散光更敏感。
- 近眼显示还要额外关注 eyebox 内的角度光谱漂移。
